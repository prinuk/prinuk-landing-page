# פרינוּק - המכירה השבועית

דף נחיתה + טופס הזמנות עבור המכירה השבועית של פרינוּק.

- **דף נחיתה** (`prinuk.co.il`) — מידע על המכירה, פרטי קשר, הזמנה מהירה בוואטסאפ
- **טופס הזמנות** (`order.prinuk.co.il`) — טופס הזמנה מלא עם קטלוג מוצרים מתוך Google Sheets

---

## מה צריך לפני שמתחילים

- חשבון GitHub (כבר יש לכם — הקוד נמצא שם)
- חשבון Google (כבר יש לכם — הגיליון נמצא שם)
- גישה לניהול DNS של הדומיין `prinuk.co.il`

---

## שלב 1: פתיחת חשבון Vercel

1. נכנסים לאתר: https://vercel.com
2. לוחצים **Sign Up**
3. בוחרים **Continue with GitHub**
4. מאשרים את ההרשאות — Vercel תבקש גישה לקרוא את הריפוזיטוריז שלכם ב-GitHub
5. אחרי שנכנסתם, תגיעו לדשבורד של Vercel

---

## שלב 2: הגדרת Google Cloud ו-Service Account

### 2.1 יצירת פרויקט ב-Google Cloud

1. נכנסים ל: https://console.cloud.google.com
2. בפינה השמאלית למעלה ליד הלוגו של Google Cloud יש בורר פרויקטים — לוחצים עליו
3. לוחצים **New Project** (פרויקט חדש)
4. נותנים שם, למשל: `prinuk`
5. לוחצים **Create**
6. מחכים כמה שניות עד שהפרויקט נוצר, ואז בוחרים אותו מהבורר

### 2.2 הפעלת Google Sheets API

1. בתוך הפרויקט, נכנסים לתפריט הצדדי (שלוש פסים למעלה משמאל) > **APIs & Services** > **Library**
2. בשורת החיפוש כותבים: `Google Sheets API`
3. לוחצים על התוצאה **Google Sheets API**
4. לוחצים **Enable** (הפעלה)

### 2.3 יצירת Service Account

1. נכנסים לתפריט הצדדי > **IAM & Admin** > **Service Accounts**
2. לוחצים **Create Service Account** למעלה
3. נותנים שם, למשל: `prinuk-sheets`
4. לוחצים **Create and Continue**
5. בשלב ה-Role (תפקיד) — אפשר לדלג, לוחצים **Continue**
6. לוחצים **Done**

### 2.4 הורדת קובץ מפתח JSON

1. ברשימת ה-Service Accounts, לוחצים על השורה של ה-Service Account שיצרתם
2. עוברים ללשונית **Keys**
3. לוחצים **Add Key** > **Create new key**
4. בוחרים **JSON**
5. לוחצים **Create**
6. קובץ JSON יורד למחשב — **שמרו אותו, תצטרכו אותו בשלב 4**

### 2.5 העתקת כתובת המייל של ה-Service Account

1. חוזרים לדף ה-Service Accounts
2. מעתיקים את כתובת המייל שנראית ככה: `prinuk-sheets@prinuk-XXXXX.iam.gserviceaccount.com`
3. **שמרו את הכתובת הזו, תצטרכו אותה בשלב 3**

## שלב 3: שיתוף הגיליון עם ה-Service Account

1. פותחים את קובץ ה-Google Sheets של פרינוק (הקובץ עם הגיליונות: מוצרים, הגדרות, הזמנות, פריטי הזמנות, דפי ליקוט)
2. לוחצים על כפתור **שיתוף** (Share) בפינה הימנית העליונה
3. מדביקים את כתובת המייל של ה-Service Account (מהשלב הקודם)
4. בוחרים הרשאת **Editor** (עורך)
5. מסירים סימון מ-**Notify people** (לא צריך לשלוח מייל)
6. לוחצים **Share**

### מציאת ה-Spreadsheet ID

1. מסתכלים על הכתובת (URL) של הגיליון בדפדפן, היא נראית ככה:
   ```
   https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz/edit
   ```
2. ה-ID הוא החלק שבין `/d/` ל-`/edit` — בדוגמה: `1AbCdEfGhIjKlMnOpQrStUvWxYz`
3. **שמרו את ה-ID הזה, תצטרכו אותו בשלב 4**

## שלב 4: העלאה ל-Vercel

לפני שמחברים את הדומיינים של הפרודקשן, מומלץ להגדיר סביבת בדיקות מרוחקת.
ראו: [REMOTE_TESTING.md](REMOTE_TESTING.md).

### 4.1 ייבוא הפרויקט

1. נכנסים לדשבורד של Vercel: https://vercel.com/dashboard
2. לוחצים **Add New** > **Project**
3. מוצאים את הריפו `prinuk-landing-page` ולוחצים **Import**
4. **לפני שלוחצים Deploy** — צריך להוסיף את משתני הסביבה (ראו 4.2)

### 4.2 הוספת משתני סביבה (Environment Variables)

בדף ה-Import, פותחים את הסקשן **Environment Variables** ומוסיפים שני משתנים:

**משתנה ראשון:**
- **Name:** `SPREADSHEET_ID`
- **Value:** ה-ID שהעתקתם בשלב 3 (למשל `1AbCdEfGhIjKlMnOpQrStUvWxYz`)

**משתנה שני:**
- **Name:** `GOOGLE_CREDENTIALS`
- **Value:** פותחים את קובץ ה-JSON שהורדתם בשלב 2.4 עם עורך טקסט (Notepad / TextEdit), מעתיקים את **כל התוכן** ומדביקים כאן

### 4.3 פריסה

1. לוחצים **Deploy**
2. מחכים עד שהפריסה מסתיימת (בדרך כלל פחות מדקה)
3. Vercel ייתן לכם כתובת זמנית כמו `prinuk-landing-page-XXXX.vercel.app` — אפשר לבדוק שהדף נחיתה עובד

## שלב 5: הגדרת דומיינים

### 5.1 הוספת דומיינים ב-Vercel

1. בדשבורד של Vercel, נכנסים לפרויקט
2. הולכים ל-**Settings** > **Domains**
3. מוסיפים את הדומיין: `prinuk.co.il` ולוחצים **Add**
4. מוסיפים את הדומיין: `order.prinuk.co.il` ולוחצים **Add**
5. Vercel יראה לכם בדיוק אילו רשומות DNS צריך להוסיף (בדרך כלל רשומת `A` או `CNAME`)

### 5.2 עדכון DNS

1. נכנסים לפאנל ניהול ה-DNS של הדומיין `prinuk.co.il` (אצל ספק הדומיין שלכם)
2. **מוחקים** את רשומות ה-DNS הישנות שהפנו ל-GitHub Pages (בדרך כלל רשומות `A` או `CNAME`)
3. **מוסיפים** את הרשומות שמופיעות בדף ה-Domains של Vercel:
   - עבור `prinuk.co.il` — בדרך כלל רשומת `A` שמפנה ל-`76.76.21.21`
   - עבור `order.prinuk.co.il` — בדרך כלל רשומת `CNAME` שמפנה ל-`cname.vercel-dns.com`
4. שינויי DNS יכולים לקחת עד 24 שעות (בדרך כלל הרבה פחות)

### 5.3 בדיקה

1. נכנסים ל-`prinuk.co.il` — צריך להופיע דף הנחיתה
2. נכנסים ל-`order.prinuk.co.il` — צריך להופיע טופס ההזמנות עם הקטלוג מהגיליון

## שלב 6: הגדרת שליחת מיילים אוטומטית

כשלקוח שולח הזמנה, Vercel כותב אותה לגיליון עם סטטוס "חדש". הסטטוס הזה מסמן שההזמנה עוד לא טופלה, והוא לא משתנה בגלל שליחת מיילים. שליחת המיילים מנוהלת רק דרך העמודות:

- `סטטוס מייל לקוח`
- `שגיאת מייל לקוח`
- `סטטוס מייל פרינוק`
- `שגיאת מייל פרינוק`
- `סטטוס טלגרם פרינוק`
- `שגיאת טלגרם פרינוק`

כדי שיישלחו מיילים והתראות טלגרם (התראה לבעל העסק + עותק ללקוח + דף ליקוט), צריך להוסיף פונקציה ב-Apps Script שרצה כל דקה ומטפלת בהזמנות שממתינות לשליחה.

### 6.1 הוספת הקוד ל-Apps Script

1. פותחים את הגיליון ב-Google Sheets
2. הולכים ל-**Extensions** > **Apps Script**
3. ליד **Files** בצד שמאל, לוחצים על הפלוס (**+**) > **Script**
4. נותנים לקובץ את השם: `emailTrigger`
5. מוחקים את כל מה שכתוב בקובץ החדש
6. פותחים את הקובץ `apps-script/email-trigger.gs` מתוך הפרויקט
7. מעתיקים את **כל התוכן** ומדביקים בקובץ החדש ב-Apps Script
8. לוחצים **Ctrl+S** (או **Cmd+S**) לשמירה

### 6.2 הגדרת טריגר (הרצה אוטומטית כל דקה)

1. ב-Apps Script, לוחצים על סמל השעון בצד שמאל (**Triggers**)
2. לוחצים **Add Trigger** (הוספת טריגר) בפינה הימנית התחתונה
3. ממלאים את ההגדרות הבאות:
   - **Choose which function to run:** `processNewOrders`
   - **Choose which deployment should run:** `Head`
   - **Select event source:** `Time-driven`
   - **Select type of time based trigger:** `Minutes timer`
   - **Select minute interval:** `Every minute`
4. לוחצים **Save**
5. Google תבקש הרשאות — לוחצים **Review permissions** > בוחרים את החשבון > **Allow**

### 6.3 בדיקה

1. חוזרים לעורך הקוד ב-Apps Script
2. בוחרים את הפונקציה `processNewOrders` מהתפריט למעלה
3. לוחצים **Run** (הפעלה)
4. אם יש הזמנות שממתינות לשליחת מייל, המיילים יישלחו
5. העמודות של סטטוס המייל יתעדכנו ל-`נשלח ל-Google`, `נשלח ללא לוגו`, `לא נמסר מייל`, או `נכשל`
6. העמודה `סטטוס` נשארת `חדש` עד שמטפלים בהזמנה ידנית

### מה הפונקציה עושה:

- מחפשת הזמנות לפי עמודות סטטוס המייל, לא לפי העמודה `סטטוס`
- לכל הזמנה שממתינה לשליחה:
  - שולחת מייל התראה לבעל העסק (עם PDF מצורף)
  - שולחת עותק מייל ללקוח (אם מסר מייל)
  - מעדכנת את דף הליקוט
  - שולחת התראת טלגרם על הזמנה חדשה אם הוגדר בוט
  - שולחת התראת טלגרם דחופה אם שליחת מייל נכשלה סופית
  - מנסה שוב עד 3 פעמים במקרה של תקלה זמנית
  - לא משנה את העמודה `סטטוס`

### 6.4 הגדרת התראות Telegram

1. פותחים Telegram ומחפשים את `@BotFather`
2. שולחים `/newbot`, בוחרים שם לבוט ושם משתמש שמסתיים ב-`bot`
3. מעתיקים את ה-token ש-BotFather נותן
4. יוצרים קבוצת Telegram לבעלי העסק ומוסיפים אליה את הבוט
5. שולחים הודעה אחת בקבוצה
6. פותחים בדפדפן:
   ```
   https://api.telegram.org/bot<BOT_TOKEN>/getUpdates
   ```
7. מעתיקים את הערך `chat.id` של הקבוצה
8. בגיליון `הגדרות`, ממלאים:
   - `טלגרם בוט טוקן` — ה-token מ-BotFather
   - `טלגרם צ׳אט ID` — ה-chat ID של הקבוצה
9. מריצים מהתפריט בגיליון: `פרינוק > הכן גיליונות הזמנות`
10. מריצים מהתפריט: `פרינוק > בדיקת טלגרם`

## שלב 7: כיבוי ה-Apps Script Web App

אחרי שהכל עובד על Vercel, צריך לכבות את ה-Web App הישן:

1. פותחים את הגיליון ב-Google Sheets
2. הולכים ל-**Extensions** > **Apps Script**
3. הולכים ל-**Deploy** > **Manage deployments**
4. ליד ה-deployment הפעיל, לוחצים על הסמל של עריכה (עיפרון)
5. משנים את הסטטוס ל-**Archived**
6. לוחצים **Deploy**

**חשוב:** אל תמחקו את הקוד של ה-Apps Script! הפונקציות של התפריט (רענון מוצרים, יצירת PDF, ארכיון הזמנות) ממשיכות לעבוד כרגיל. גם הטריגר של המיילים שהגדרתם בשלב 6 ממשיך לרוץ.

## ניהול שוטף

### הכל נשאר אותו דבר:
- **עדכון מוצרים** — בגיליון "מוצרים" (או דרך "רענן מוצרים מחישוב מחירים" מהתפריט)
- **הגדרות טופס** — בגיליון "הגדרות" (כותרת, תיאור, פרטי איסוף, לוגו, טלפון, מייל)
- **צפייה בהזמנות** — בגיליון "הזמנות" ו-"פריטי הזמנות"
- **ארכיון ו-PDF** — מהתפריט "פרינוק" בגיליון
- **מיילים** — נשלחים אוטומטית תוך דקה מרגע ההזמנה

### מה שונה:
- הטופס מוגש מ-Vercel במקום מ-Apps Script — יותר מהיר ויציב בזמן מכירות
- הקטלוג נשמר בזיכרון מטמון (cache) ל-60 שניות — אחרי שינוי מוצרים, לוקח עד דקה עד שהטופס מתעדכן
- מיילים נשלחים תוך דקה (במקום מיד) — כי הטריגר רץ כל דקה
